# EM8 命令セットリファレンス

## 1. 概要

EM8 はカスタム 8 ビット CPU で、**MOS 6502 命令セットを直接モデル
としています**。レジスタ (`A`, `X`, `Y`)、フラグレイアウト、
`$0100-$01FF` のスタックページ、リセット / NMI / IRQ ベクタ
(`$FFFC-$FFFD` / `$FFFA-$FFFB` / `$FFFE-$FFFF`)、命令エンコーディング
(例: `$A9` `LDA #imm`、`$20` `JSR`、`$60` `RTS`、`$EA` `NOP`) は
すべて NMOS 6502 とバイト単位で一致します。

EM8 は **10 進 (BCD) モード** (`D` フラグ、`CLD` / `SED` ともに削除)
および **間接インデックス系アドレッシングモード** (`(zp,X)` /
`(zp),Y` / `LDX $nn,Y`) を省略しています。一方、利便性のため
65C02 由来の `BRA` (常時分岐) と、エミュレータ制御用の `HLT` 命令を
追加しています。対応する 6502 サブセットを対象とするプログラムや
ツールチェインは、そのままアセンブル・実行できます。本プラグインは
`emfe` C ABI を検証するための、小規模で十分に枯れたリファレンス
ターゲットとして整備されています。

主な特徴:

- 8 ビットデータバス、16 ビットアドレスバス
- 64KB リニアアドレス空間 ($0000-$FFFF)
- 3 本の 8 ビット汎用レジスタ (A, X, Y)
- ハードウェアスタック $0100-$01FF (256 バイト、下方向に伸長)
- メモリマップド I/O $F000-$F03F
- ベクタ方式割り込み (IRQ, NMI, Reset)

## 2. レジスタ

| レジスタ | 幅 | 説明 |
|---------|-----|------|
| A | 8 ビット | アキュムレータ。算術・論理演算および I/O の主要レジスタ。 |
| X | 8 ビット | インデックスレジスタ X。インデックスアドレッシングやループカウンタに使用。 |
| Y | 8 ビット | インデックスレジスタ Y。インデックスアドレッシングに使用。 |
| SP | 8 ビット | スタックポインタ。スタックページ $0100-$01FF 内のオフセット。リセット時 $FF に初期化。 |
| PC | 16 ビット | プログラムカウンタ。次に実行する命令のアドレスを指す。 |
| FL | 8 ビット | フラグレジスタ。プロセッサステータスフラグを格納。 |

## 3. フラグレジスタ (FL)

フラグレジスタは 8 ビット幅です。各ビットの割り当ては以下の通りです。

```
  Bit 7   Bit 6   Bit 5   Bit 4   Bit 3   Bit 2   Bit 1   Bit 0
+-------+-------+-------+-------+-------+-------+-------+-------+
|   N   |   V   |   -   |   B   |   -   |   I   |   Z   |   C   |
+-------+-------+-------+-------+-------+-------+-------+-------+
```

| ビット | 名前 | 説明 |
|-------|------|------|
| 7 | N (Negative) | 演算結果のビット 7 がセットされている場合にセット (2 の補数で負)。 |
| 6 | V (Overflow) | 符号付き算術演算でオーバーフローが発生した場合にセット。 |
| 5 | - | 未使用、常に 1。 |
| 4 | B (Break) | BRK 命令による割り込みの場合にセット。スタック上の FL コピーにのみ存在。 |
| 3 | - | 未使用、常に 1。 |
| 2 | I (Interrupt Disable) | セットされている場合、IRQ 割り込みがマスクされる。NMI は影響を受けない。 |
| 1 | Z (Zero) | 演算結果がゼロの場合にセット。 |
| 0 | C (Carry) | 符号なし算術演算でキャリーまたはボローが発生した場合にセット。シフト・ローテート命令でも使用。 |

## 4. メモリマップ

| アドレス範囲 | サイズ | 説明 |
|-------------|--------|------|
| $0000-$00FF | 256 バイト | ゼロページ。ゼロページアドレッシングモードによる高速アクセス。 |
| $0100-$01FF | 256 バイト | ハードウェアスタック。SP がこのページ内をインデックス。 |
| $0200-$EFFF | 60,928 バイト | 汎用 RAM。プログラムコードおよびデータ。 |
| $F000-$F03F | 64 バイト | メモリマップド I/O ペリフェラル。 |
| $F040-$FFF9 | 4,026 バイト | 予約 / 追加 RAM。 |
| $FFFA-$FFFB | 2 バイト | NMI ベクタ (リトルエンディアン)。 |
| $FFFC-$FFFD | 2 バイト | リセットベクタ (リトルエンディアン)。 |
| $FFFE-$FFFF | 2 バイト | IRQ/BRK ベクタ (リトルエンディアン)。 |

## 5. アドレッシングモード

| モード | 構文 | 説明 | 例 |
|--------|------|------|-----|
| Implied | (なし) | オペランドは命令に暗黙的に含まれる。 | `INX` |
| Immediate | `#$nn` | オペコード直後の 8 ビット即値。 | `LDA #$42` |
| Zero Page | `$nn` | ゼロページ ($0000-$00FF) 内の 8 ビットアドレス。 | `LDA $10` |
| Absolute | `$nnnn` | 完全な 16 ビットアドレス。 | `LDA $1234` |
| Zero Page,X | `$nn,X` | ゼロページアドレス + X レジスタ (ゼロページ内でラップ)。 | `LDA $10,X` |
| Absolute,X | `$nnnn,X` | 絶対アドレス + X レジスタ。 | `LDA $1234,X` |
| Absolute,Y | `$nnnn,Y` | 絶対アドレス + Y レジスタ。 | `LDA $1234,Y` |
| Indirect | `($nnnn)` | 指定アドレスから読み取った 16 ビットアドレス (JMP のみ)。 | `JMP ($1234)` |
| Relative | `$nn` (符号付き) | PC+2 からの符号付き 8 ビットオフセット (分岐命令用)。 | `BEQ label` |

全分岐命令は 2 バイト: オペコード + 符号付き 8 ビット相対オフセット。オフセットは分岐命令フェッチ後の PC (PC+2) からの相対値。分岐不成立時 2 サイクル、成立時 3 サイクル、ページ境界をまたぐ場合 +1 サイクル。

| $D0 | BNE | Z = 0 | ゼロでなければ分岐 (等しくない) |
| $B0 | BCS | C = 1 | キャリーセットなら分岐 |
| $90 | BCC | C = 0 | キャリークリアなら分岐 |

## 7. 割り込み機構

### 7.1 IRQ (割り込み要求)

ペリフェラルデバイスによるハードウェア割り込み。I フラグでマスク可能。

1. 現在の命令が完了する。
2. I フラグがクリア (割り込み有効) の場合:
   a. PC 上位バイトをスタックにプッシュ。
   b. PC 下位バイトをスタックにプッシュ。
   c. FL をスタックにプッシュ (プッシュされるコピーの B フラグはクリア)。
   d. I フラグをセット (以降の IRQ を無効化)。
   e. IRQ ベクタ $FFFE-$FFFF (リトルエンディアン) から PC をロード。

### 7.2 NMI (マスク不可割り込み)

NMI 入力エッジでトリガされるマスク不可割り込み。I フラグの影響を受けない。

1. 現在の命令が完了する。
2. PC 上位バイトをスタックにプッシュ。
3. PC 下位バイトをスタックにプッシュ。
4. FL をスタックにプッシュ (プッシュされるコピーの B フラグはクリア)。
5. I フラグをセット。
6. NMI ベクタ $FFFA-$FFFB (リトルエンディアン) から PC をロード。

### 7.3 BRK (ソフトウェア割り込み)

BRK 命令 ($00) によるソフトウェアトリガの割り込み。

1. PC を 1 インクリメント (BRK にはパディングバイトがある)。
2. PC 上位バイトをスタックにプッシュ。
3. PC 下位バイトをスタックにプッシュ。
4. FL をスタックにプッシュ (B フラグをセット)。
5. I フラグをセット。
6. IRQ ベクタ $FFFE-$FFFF (リトルエンディアン) から PC をロード。

ISR はスタック上の FL の B フラグを調べることで、BRK とハードウェア IRQ を区別できます。

### 7.4 RTI (割り込みからのリターン)

RTI 命令は割り込みエントリシーケンスを逆転します:

1. スタックから FL をプル (I フラグを含む全フラグを復元)。
2. スタックから PC 下位バイトをプル。
3. スタックから PC 上位バイトをプル。
4. 復元された PC で実行を再開。

### 7.5 割り込み優先度

| 優先度 | ソース | ベクタ | マスク可能 |
|--------|--------|--------|-----------|
| 最高 | Reset | $FFFC | いいえ |
| 高 | NMI | $FFFA | いいえ |
| 低 | IRQ/BRK | $FFFE | はい (I フラグ) |

## 8. ペリフェラルレジスタ

全ペリフェラルは I/O 領域 $F000-$F03F にメモリマップされています。

### 8.1 UART ($F000-$F00F)

シリアルキャラクタ I/O 用のシンプルな UART。

| アドレス | 名前 | R/W | 説明 |
|---------|------|-----|------|
| $F000 | UART_DATA | R/W | **リード**: 受信データレジスタ。RX FIFO から次の文字を返す。**ライト**: 送信データレジスタ。TX 出力に文字を送信。 |
| $F001 | UART_STATUS | R | ステータスレジスタ (下記参照)。 |
| $F002 | UART_CONTROL | R/W | コントロールレジスタ (下記参照)。 |

**UART_STATUS ($F001) ビット:**

| ビット | 名前 | 説明 |
|-------|------|------|
| 0 | RX_READY | 1 = RX FIFO にデータあり |
| 1 | TX_READY | 1 = 送信機がデータ受付可能 |
| 7-2 | - | 予約、0 として読み出される |

**UART_CONTROL ($F002) ビット:**

| ビット | 名前 | 説明 |
|-------|------|------|
| 0 | RX_IRQ_EN | 1 = RX_READY セット時に IRQ を有効化 |
| 1 | TX_IRQ_EN | 1 = TX_READY セット時に IRQ を有効化 |
| 7-2 | - | 予約、0 を書き込むこと |

### 8.2 タイマー ($F010-$F01F)

プログラマブルインターバルタイマー。リロード値からカウントダウンし、アンダーフロー時に IRQ を生成。

| アドレス | 名前 | R/W | 説明 |
|---------|------|-----|------|
| $F010 | TMR_COUNT_LO | R | 現在のカウンタ値、下位バイト。 |
| $F011 | TMR_COUNT_HI | R | 現在のカウンタ値、上位バイト。 |
| $F012 | TMR_RELOAD_LO | R/W | リロード値、下位バイト。 |
| $F013 | TMR_RELOAD_HI | R/W | リロード値、上位バイト。 |
| $F014 | TMR_CONTROL | R/W | タイマーコントロールレジスタ (下記参照)。 |
| $F015 | TMR_STATUS | R | タイマーステータスレジスタ (下記参照)。 |

**TMR_CONTROL ($F014) ビット:**

| ビット | 名前 | 説明 |
|-------|------|------|
| 0 | ENABLE | 1 = タイマー動作中 |
| 1 | IRQ_EN | 1 = アンダーフロー時に IRQ を生成 |
| 2 | AUTO_RELOAD | 1 = アンダーフロー時にカウンタをリロード (周期モード)。0 = ワンショット。 |
| 7-3 | - | 予約、0 を書き込むこと |

**TMR_STATUS ($F015) ビット:**

| ビット | 名前 | 説明 |
|-------|------|------|
| 0 | UNDERFLOW | 1 = カウンタがアンダーフロー。1 を書き込むとクリア。 |
| 7-1 | - | 予約、0 として読み出される |

タイマーは CPU サイクルごとに 1 カウント減少します。ENABLE セット時にカウンタが 0 に達すると:

1. TMR_STATUS の UNDERFLOW フラグがセットされる。
2. IRQ_EN がセットされている場合、IRQ がアサートされる。
3. AUTO_RELOAD がセットされている場合、TMR_RELOAD からカウンタがリロードされる。そうでなければタイマーは停止。

### 8.3 GPIO ($F020-$F02F)

8 ビット汎用 I/O ポート。

| アドレス | 名前 | R/W | 説明 |
|---------|------|-----|------|
| $F020 | GPIO_DATA | R/W | **リード**: 現在のピンレベル。**ライト**: 出力ラッチ。 |
| $F021 | GPIO_DIR | R/W | 方向レジスタ。1 = 出力、0 = 入力。デフォルト: $00 (全入力)。 |
| $F022 | GPIO_IRQ_EN | R/W | ピンごとの IRQ 有効化。1 = ピン変化時に IRQ を有効化 (入力ピンのみ)。 |
| $F023 | GPIO_IRQ_FLAG | R/W | ピンごとの IRQ フラグ。1 = ピンがトリガ。1 を書き込むとクリア。 |

### 8.4 割り込みコントローラ ($F030-$F03F)

グローバル割り込みステータスおよび制御。

| アドレス | 名前 | R/W | 説明 |
|---------|------|-----|------|
| $F030 | IRQEN | R/W | 割り込み有効化レジスタ (下記参照)。 |
| $F031 | IRQFLAG | R | 割り込みペンディングフラグ (下記参照)。 |
| $F032 | IRQACK | W | 割り込みアクノリッジ。1 ビットを書き込むと対応する IRQFLAG ビットをクリア。 |

**IRQEN / IRQFLAG ビット割り当て:**

| ビット | ソース | 説明 |
|-------|--------|------|
| 0 | UART_RX | UART 受信データあり |
| 1 | UART_TX | UART 送信機準備完了 |
| 2 | TIMER | タイマーアンダーフロー |
| 3 | GPIO | GPIO ピン変化 |
| 7-4 | - | 予約 |

`(IRQFLAG & IRQEN) != 0` かつ CPU の I フラグがクリアの場合に、IRQ が CPU にアサートされます。

## 9. リセット動作

CPU がリセットされると (電源投入または明示的リセット):

1. I フラグがセットされる (割り込み無効)。
2. SP が $FF に初期化される。
3. レジスタ A, X, Y が $00 にセットされる。
4. FL の未使用ビット (5 と 3) が 1 にセットされる。
5. リセットベクタ $FFFC-$FFFD (リトルエンディアン) から PC がロードされる。
6. ロードされた PC アドレスから実行を開始する。
7. 全ペリフェラルレジスタがデフォルト値にリセットされる。
